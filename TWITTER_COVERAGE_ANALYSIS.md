## 根本原因分析

### 为什么覆盖率这么低？

可能的原因：
1. **新代币太新**: 推特搜索需要时间索引
2. **搜索查询单一**: 仅搜索合约地址，可能遗漏其他关键词
3. **平台特性**: four.meme平台的代币可能不活跃于Twitter
4. **API限制**: 可能有部分搜索失败但未记录

## 对因子使用的影响

### 严重程度评估

| 影响类型 | 严重程度 | 说明 |
|---------|---------|------|
| 统计可靠性 | 🔴 极高 | 基于极小样本的结论不可靠 |
| 因子可用性 | 🟡 中等 | 只有13.9%的代币有推特因子值 |
| 策略信号 | 🟠 较高 | 大部分代币无法获得推特信号 |
| 偏差风险 | 🟢 低 | 各质量组覆盖率相似，无选择偏差 |

### 实际影响

1. **因子值缺失**: 86.1%的代币推特因子为0或null
2. **信号不完整**: 买入决策可能缺少重要信息
3. **回测偏差**: 历史数据中推特因子大量缺失
4. **统计失效**: 无法建立可靠的质量预测模型

## 建议

### 短期建议

1. **降低推特因子权重**
   - 覆盖率仅13.9%，不应作为主要决策依据
   - 可作为辅助参考（有数据时）

2. **标记缺失状态**
   ```javascript
   // 区分"无推文"和"无数据"
   factors: {
     twitter_followers: 0,
     twitter_data_available: false,  // 新增标记
     twitter_search_attempted: true
   }
   ```

3. **避免过度解读**
   - 之前的"中质量粉丝数高"结论样本量太小
   - 高质量仅1个样本，不具备统计意义

### 长期建议

1. **改进数据采集**
   - 增加搜索关键词（代币符号、名称）
   - 尝试多个Twitter API
   - 延迟搜索（等待Twitter索引）

2. **数据源多样化**
   - 考虑其他社交平台（Telegram, Discord）
   - 链上数据分析（持有人分布）
   - 交易数据分析（交易量、持仓时间）

3. **因子设计调整**
   - 设计"数据缺失率"作为风险指标
   - 对缺失数据采用保守策略（不是0，而是"未知"）
   - 使用贝叶斯方法处理缺失值
