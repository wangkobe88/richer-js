三、检测效果对比
================================================================================

【使用全部交易】
  blockGapP75 < 3:
  - 拉砸拒绝: 80% (8/10)
  - 正常通过: 71.4% (5/7)

【去掉小额交易 (<$10)】
  blockGapP75 < 3:
  - 拉砸拒绝: 70% (7/10)
  - 正常通过: 71.4% (5/7)

结论: 去掉小额交易后，拉砸拒绝率略降，但正常通过率不变。

================================================================================
五、关键发现与建议
================================================================================

1. **小额交易占比本身就是一个很好的区分因子**:
   - 拉砸代币: 平均 27.8%
   - 正常代币: 平均 17.1%
   - 建议: 添加 low_value_trade_ratio 因子

2. **去掉小额交易后的检测效果**:
   - blockGapP75 仍然有效
   - 但拉砸拒绝率略降 (80% → 70%)
   - 可能需要调整阈值

3. **组合建议**:
   方案A: 使用全部交易
     block_gap_p75 < 3

   方案B: 结合小额交易占比
     (block_gap_p75 < 3) OR (low_value_trade_ratio > 0.25 AND block_gap_p75 < 4)

   方案C: 结合钱包簇
     block_gap_p75 < 3 AND wallet_cluster_second_to_first_ratio < 0.3

================================================================================
六、新增因子建议
================================================================================

建议添加以下因子到 EarlyParticipantCheckService:

1. low_value_trade_ratio
   - 定义: USD < $10 的交易占比
   - 计算: count(value < 10) / total_trades
   - 预期: 拉砸代币 > 0.25, 正常代币 < 0.20

2. block_gap_p75 (使用全部交易)
   - 定义: 区块号间隔的75分位数
   - 预期: 拉砸代币 < 3, 正常代币 >= 3

3. block_gap_mean (使用全部交易)
   - 定义: 区块号间隔平均值
   - 预期: 拉砸代币 < 1.5, 正常代币 >= 1.5
