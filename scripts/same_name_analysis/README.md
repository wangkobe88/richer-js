# 同名代币分析报告

## 分析背景

本分析旨在研究同名代币（相同symbol的代币）与代币质量之间的关联性，为购买前检查提供新的因子参考。

**实验ID**: 25493408-98b3-4342-a1ac-036ba49f97ee
**分析时间**: 2025-03-11
**代币样本**: 63个有购买信号的代币（高质量8个，中质量26个，低质量29个）

## 分析方法

### AVE API搜索

使用AVE API的`searchTokens`接口搜索与目标代币相同symbol的所有代币：

```javascript
const aveAPI = new AveTokenAPI('https://prod.ave-api.com', 30000, AVE_API_KEY);
const results = await aveAPI.searchTokens(symbol, null, 300, 'fdv');
```

### 两种匹配方式对比

#### 方式1：仅按Symbol匹配（原始版本）

仅根据symbol搜索，不过滤名称。问题：很多symbol相同但name完全不同的代币也被计入。

#### 方式2：严格名称匹配（改进版本）

**匹配规则**：
1. name 完全相同（忽略大小写和空格）
2. name 互相包含（处理 "Leo" vs "Leo Token" 的情况）
3. symbol 相同且 name 包含相同的核心词

**过滤效果**：
- 总过滤率：26.0%
- 被过滤代币数：2162个（从8324个搜索结果中）

**典型案例**：
- "COS" symbol：搜索300个，过滤265个（88.3%），保留35个
- "BINANCE" symbol：搜索300个，过滤198个（66%），保留102个
- "鱼缸" symbol：搜索14个，过滤1个（7.1%），保留13个（是真同名）

### 分析指标

- **同名代币数量**: 搜索结果总数（含自己）
- **排除自己的同名数量**: 搜索结果减去自己
- **最大FDV**: 同名代币中的最大完全稀释市值
- **最大市值**: 同名代币中的最大市值
- **高市值同名**: 是否存在FDV > $1M的同名代币
- **超高市值同名**: 是否存在FDV > $10M的同名代币

### 阈值设置

- **高FDV阈值**: $1,000,000 (100万美元)
- **超高FDV阈值**: $10,000,000 (1000万美元)

## 核心发现

### 0. 严格匹配的有效性

**问题发现**：仅按symbol搜索会包含大量"假同名"代币：
- "LEO" symbol：包含 "DEMO-CAT"、"Leo Messi" 等不同名称的代币
- "CTO" symbol：包含 "Chinese Take Over"、"Cats Take Over" 等完全不同的代币
- "1$" symbol：包含 "My Last Dollar"、"Charlie Kirk Coin" 等

**改进效果**：严格名称匹配过滤掉 **26.0%** 的虚假同名代币。

### 1. 同名代币覆盖率

```
总代币数: 63
有同名代币(排除自己): 62 (98.4%)
有严格同名(排除自己): 61 (96.8%)
```

**几乎所有的代币都有同名代币**，这意味着"是否有同名代币"本身没有区分度。

### 2. 同名代币数量分布

| 范围 | 数量 | 占比 |
|------|------|------|
| 0个同名 | 0 | 0.0% |
| 1-5个 | 2 | 3.2% |
| 6-10个 | 4 | 6.3% |
| 11-20个 | 5 | 7.9% |
| **20+个** | **52** | **82.5%** |

### 3. 质量区分度 - 同名代币数量

#### 版本1：仅按Symbol匹配

| 质量等级 | 平均同名数 | 中位数 | 11+个占比 |
|---------|-----------|--------|----------|
| 高质量 | 163.5 | **141** | 100% |
| 中质量 | 158.7 | **142** | 92.3% |
| 低质量 | 98.2 | **25** | 86.2% |

#### 版本2：严格名称匹配

| 质量等级 | 平均同名数 | 中位数 | 11+个占比 |
|---------|-----------|--------|----------|
| 高质量 | 128.6 | **118** | 100% |
| 中质量 | 119.0 | **88** | 92.3% |
| 低质量 | 70.4 | **24** | 75.9% |

**关键发现**:
- **严格匹配后区分度略有改善**：高质量/低质量 中位数比从 5.6x → **4.9x**
- 高质量代币100%都有11+个严格同名代币，低质量仅75.9%
- **同名代币数量具有一定区分度**

### 4. 质量区分度 - 最大FDV

| 质量等级 | 平均最大FDV | 中位数最大FDV | 有高市值同名占比 |
|---------|------------|--------------|----------------|
| 高质量 | $552.70M | **$78.7K** | 37.5% |
| 中质量 | $71,665.34M | **$59.7K** | 42.3% |
| 低质量 | $301,396.75M | **$5.8K** | 31.0% |

**关键发现**:
- **平均值被极端值严重拉高**（见下文"极端值分析"）
- 使用中位数更可靠：高质量是低质量的 **13.5倍**
- 但"有高市值同名"的占比差异仅 **6.5%**，区分度不明显

### 5. 极端值影响

低质量组中存在多个极端值，严重拉高了平均最大FDV：

| Symbol | 同名数量 | 最大FDV | 原因 |
|--------|---------|---------|------|
| 1$ | 300 | $4.65T | 美元symbol |
| $250 | 300 | $3.29T | 常见金额符号 |
| CTO | 300 | $384B | 常见缩写 |
| Lenny | 300 | $318B | 常见名字 |

**这些极端值说明**：
- symbol的常见性比代币质量更能影响同名代币的FDV
- 使用"最大FDV"作为风险指标不可靠
- 应使用中位数而非平均值来评估

### 6. 按质量分组的同名数量分布

| 范围 | 高质量 | 中质量 | 低质量 |
|------|--------|--------|--------|
| 0个 | 0/8 (0%) | 0/26 (0%) | 0/29 (0%) |
| 1-5个 | 0/8 (0%) | 1/26 (3.8%) | 1/29 (3.4%) |
| 6-10个 | 0/8 (0%) | 1/26 (3.8%) | 3/29 (10.3%) |
| **11+个** | **8/8 (100%)** | **24/26 (92.3%)** | **25/29 (86.2%)** |

高质量代币全部都有11个以上的同名代币，而低质量有13.8%少于11个。

## 结论与建议

### 核心结论（严格匹配版本）

1. **同名代币数量有中等区分度**
   - 高质量中位数：118个
   - 低质量中位数：24个
   - 区分度：**4.9倍**

2. **同名代币最大FDV区分度弱**
   - 严格匹配后，"有高市值同名"占比差异为 **-6.0%**（高质量反而更低）
   - 被常见symbol的极端值严重干扰
   - **不建议作为风险指标**

3. **严格名称匹配是必要的**
   - 过滤掉26%的虚假同名代币
   - 提高了数据的准确性

### 与其他分析对比

| 指标 | 同名代币数量 | Twitter粉丝数 | Twitter互动数 |
|-----|-------------|--------------|--------------|
| 高/低质量区分度 | 4.9倍 | 412倍 | ∞ |
| 可用性 | 中等 | **强** | **强** |

**结论**：同名代币数量的区分度远不如Twitter数据，不建议作为主要决策因子。

### 因子建议

#### 可考虑的因子（优先级低）

**`strictSameNameTokenCount`** - 严格同名代币数量
- 高质量：中位数118
- 低质量：中位数24
- 区分度：**4.9倍**

建议阈值：
- `strictSameNameTokenCount >= 50` 可能表示较高的质量
- `strictSameNameTokenCount < 10` 可能表示较低的质量

#### 不推荐的因子

- **`hasHighFDVSameName`** - 有高市值同名代币
  - 区分度差异为负值（-6.0%），不够显著

- **`maxSameNameFDV`** - 同名代币最大FDV
  - 被常见symbol极端值严重干扰
  - 与代币质量相关性弱

- **非严格匹配的`sameNameTokenCount`**
  - 包含大量虚假同名代币（26%）
  - 必须使用严格匹配

### 实施建议

#### 1. 必须使用严格名称匹配

**重要**：仅按symbol搜索会包含大量虚假同名代币，必须使用严格名称匹配。

```javascript
// 严格匹配函数示例
function isSameName(name1, symbol1, name2, symbol2) {
  const n1 = name1.toLowerCase().trim();
  const n2 = name2.toLowerCase().trim();

  // 规则1: name 完全相同
  if (n1 === n2) return true;

  // 规则2: name 互相包含
  const shorter = n1.length < n2.length ? n1 : n2;
  const longer = n1.length < n2.length ? n2 : n1;
  if (longer.includes(shorter) && shorter.length >= 3) return true;

  return false;
}

// 使用严格匹配过滤
const strictSameNameTokens = searchResults.filter(t =>
  isSameName(tokenName, tokenSymbol, t.name, t.symbol)
);
```

#### 2. 同名代币数量可作为辅助因子（优先级低）

```javascript
// 在 PreBuyCheckService 中添加
const strictSameNameTokenCount = await getStrictSameNameTokenCount(token);

// 条件示例
strictSameNameTokenCount >= 50  // 可能是正面信号
strictSameNameTokenCount < 10   // 可能是负面信号
```

#### 3. 权重建议

同名代币数量应作为**最低优先级的辅助指标**，权重远低于：
- Twitter数据（粉丝数412倍区分度）
- 早期参与者数据（交易量、钱包数）
- Holder黑名单检查

#### 4. 结合其他指标使用

```
高质量特征：
- 同名代币数量多 (>= 50)
- Twitter粉丝多
- 早期交易活跃

低质量特征：
- 同名代币数量少 (< 10)
- Twitter无或少
- 早期交易稀少
```

### 局限性

1. **样本量较小**: 仅63个代币，需要更多数据验证
2. **Symbol偏差**: 常见symbol（如1$、CTO）需要严格名称匹配过滤
3. **链差异**: 未区分不同区块链上的同名代币
4. **时间因素**: 未考虑同名代币的创建时间关系
5. **名称匹配复杂**: 严格匹配规则可能仍有边界情况未覆盖

### 后续分析建议

1. **扩大样本**: 收集更多实验数据验证
2. **链特异性分析**: 分别分析各链的同名代币情况
3. **时间序列**: 研究同名代币创建时间与质量的关系
4. **组合因子**: 将同名代币数量与其他因子组合使用
